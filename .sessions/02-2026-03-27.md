## Environment Variables Fix

**Issue:** Sanity dev server failed with "Configuration must contain projectId"

**Root Cause:** Environment variables not loading in Sanity v5

**Solution:** Per Sanity docs, variables must use `SANITY_STUDIO_` prefix

```bash
# .env file (gitignored)
SANITY_STUDIO_PROJECT_ID=qva67g1y
SANITY_STUDIO_DATASET=production
```

Sanity auto-loads `.env` files - no dotenv package needed.
